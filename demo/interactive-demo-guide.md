## ðŸ› ï¸ Troubleshooting During Demo

### Common Issues & Solutions

**Services Not Responding**
- Check `docker-compose ps` for service status
- Restart specific services: `docker-compose restart [service]`
- Check logs: `docker-compose logs [service]`

**Demo Data Missing**
- Re-run data generation: `cd demo/data-generator && node src/generate-demo-data.js`
- Verify database connectivity
- Check for data generation errors in logs

**WebSocket Notifications Not Working**
- Verify WebSocket connection in browser dev tools
- Check for CORS issues
- Restart frontend service

**Grafana Dashboards Empty**
- Verify Prometheus is collecting metrics
- Check data source configuration
- Wait 2-3 minutes for metrics to populate

### Emergency Backup Plans

**If Live Demo Fails**
1. Switch to pre-recorded video demonstration
2. Use static screenshots with detailed narration
3. Focus on architecture diagrams and business value
4. Schedule follow-up technical demonstration

**If Specific Features Don't Work**
1. Acknowledge the issue professionally
2. Explain the feature conceptually
3. Show related working features
4. Offer detailed technical follow-up
